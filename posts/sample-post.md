## How to write posts in Gridea

### Title
Your post title goes here. Keep it catchy and descriptive.

### Metadata (set in Gridea UI, not in the post)
- **Tags**: Select from existing or create new (e.g., "Linux", "Tutorial", "Tips")
- **Featured Image**: Upload a cover image (optional but recommended)
- **Publish Date**: Automatically set to today

### Content Format

You can write using:

#### Option 1: Rich Text Editor
- Uses visual formatting buttons
- Simple for non-technical users
- Best for: Blog posts with basic formatting

#### Option 2: Markdown
- Uses simple text formatting
- **Bold text**: `**bold**`
- *Italic text*: `*italic*`
- [Links](https://example.com): `[text](url)`
- Code: \`code\` or code blocks with ``\`\`\`
- Lists: Start with `-` or numbers

### Tips for Better Posts

1. **Keep it scannable**
   - Use headings to break up content
   - Use bullet points
   - Keep paragraphs short

2. **Add images**
   - Upload images from your computer
   - Use Gridea's image insertion feature
   - Helps break up text and engage readers

3. **Proofread**
   - Use the preview button before publishing
   - Check for typos and formatting

4. **Use tags consistently**
   - "Linux Concepts" not "linux-concepts"
   - "Docker Tutorial" not "docker"
   - This helps organize your content

### Publishing

1. Click **"Save"** in Gridea
2. Click **"Preview"** to check how it looks
3. Click **"Publish"** when ready
4. Gridea pushes to GitHub automatically
5. Wait 2 minutes for your site to update
6. Visit avidflick.in to see your post!
