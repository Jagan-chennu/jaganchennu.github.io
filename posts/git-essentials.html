Git is the industry standard for version control. Whether working solo or on large teams, Git helps you manage code changes effectively. This guide covers the essentials you need to know.

What is Git?

Git is a distributed version control system that tracks changes to your code over time. It allows you to collaborate with others, revert to previous versions, and create branches for new features.

Basic Configuration

$ git config --global user.name "Your Name"
$ git config --global user.email "your.email@example.com"

Essential Commands

Initialize a Repository

$ git init my-project
$ cd my-project

Check Status

$ git status

Stage Changes

$ git add file.txt       # Add specific file
$ git add .              # Add all changes

Commit Changes

$ git commit -m "Add feature X"

View History

$ git log --oneline

Branching

$ git branch feature/new-ui        # Create branch
$ git checkout feature/new-ui      # Switch to branch
$ git checkout -b feature/api      # Create and switch

$ git merge feature/new-ui         # Merge branch

Remote Repositories

$ git remote add origin https://github.com/user/repo.git
$ git push origin main             # Push to remote
$ git pull origin main             # Pull from remote

Master these commands and you'll have a solid foundation in version control!
