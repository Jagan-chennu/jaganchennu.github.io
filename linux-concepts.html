Getting Started with Linux: Essential Commands

üìÖ February 13, 2026 ‚è±Ô∏è 8 min read üìä Beginner

Linux may seem intimidating at first, but once you master a few essential commands, you'll find that working with the terminal is actually quite intuitive. In this guide, I'll walk you through the fundamental commands every Linux user should know.

Why Learn Linux Commands?

While graphical interfaces are convenient, the Linux command line (terminal or shell) is incredibly powerful. By learning basic commands, you can:

  • Navigate your file system quickly
  • Manage files and directories efficiently
  • Install software and manage systems
  • Automate repetitive tasks
  • Troubleshoot issues effectively

Essential Navigation Commands

pwd - Print Working Directory

Shows your current location in the file system:

$ pwd
/home/username/Documents

ls - List Files

Displays files and folders in the current directory:

$ ls
Downloads  Documents  Desktop  Projects

$ ls -la  # Shows hidden files and details
total 32
drwxr-xr-x  5 username group  4096 Feb 13 10:30 .
drwxr-xr-x  3 root     root   4096 Feb 10 08:15 ..

cd - Change Directory

Navigate to a different folder:

$ cd Documents        # Go to Documents folder
$ cd ..              # Go up one level
$ cd ~               # Go to home directory
$ cd /               # Go to root directory

File and Directory Operations

mkdir - Create Directory

$ mkdir my-folder
$ mkdir -p path/to/nested/folders  # Create nested directories

touch - Create Empty File

$ touch filename.txt

cp - Copy Files

$ cp source.txt destination.txt
$ cp -r folder1 folder2  # Copy directory recursively

mv - Move or Rename

$ mv oldname.txt newname.txt  # Rename
$ mv file.txt /path/to/destination/  # Move

rm - Remove Files

$ rm filename.txt
$ rm -r folder_name  # Remove directory and contents

Viewing File Contents

cat - Display File Content

$ cat filename.txt

less - View Files Interactively

$ less largefile.txt  # Press 'q' to quit

Key Takeaways

  • Master pwd, ls, and cd for navigation
  • Learn basic file operations: cp, mv, rm
  • Practice in a safe environment before managing critical files
  • Use man commandname to read command documentation

Understanding Linux File Permissions

üìÖ February 5, 2026 ‚è±Ô∏è 10 min read üìä Intermediate

File permissions are fundamental to Linux security. Whether you're a system administrator or a regular user, understanding how permissions work is crucial.

What Are File Permissions?

File permissions determine who can read (r), write (w), or execute (x) a file or directory. There are three categories of users:

  • Owner (User): The person who created the file
  • Group: A set of users with shared permissions
  • Others: Everyone else on the system

Reading Permission Notation

When you run ls -l, you see something like:

-rw-r--r--  1  john  developers  1024  Feb 5  10:30  file.txt
drwxr-xr-x  2  john  developers  4096  Feb 4  15:45  directory

Breaking down -rw-r--r--:

  • - = Regular file (d = directory, l = symlink)
  • rw- = Owner permissions (read, write, no execute)
  • r-- = Group permissions (read only)
  • r-- = Others permissions (read only)

Changing Permissions with chmod

Symbolic Method

$ chmod u+x file.txt        # Add execute for owner
$ chmod g+w file.txt        # Add write for group
$ chmod o-r file.txt        # Remove read for others
$ chmod a+r file.txt        # Add read for all

Numeric Method

4 = read (r)
2 = write (w)
1 = execute (x)

$ chmod 755 directory       # rwx r-x r-x
$ chmod 644 file.txt        # rw- r-- r--
$ chmod 777 shared-folder   # rwx rwx rwx

Changing Ownership with chown

$ chown newowner file.txt           # Change owner
$ chown newowner:newgroup file.txt  # Change owner and group
$ chown -R newowner directory       # Change recursively

Best Practices

  • Never give unnecessary permissions to "Others"
  • Use group permissions to share access securely
  • Regularly audit file permissions on critical files
  • Use 644 for regular files, 755 for executables
  • Be careful with chmod 777 - it's a security risk

Real-World Scenarios

Scenario: Protect Sensitive Configuration File

$ chmod 600 /etc/myapp/secrets.conf  # Owner read/write only
$ chown myapp:myapp /etc/myapp/secrets.conf

Scenario: Share Script with Team

$ chmod 755 deploy.sh              # Owner: rwx, Others: r-x
$ chown developer:team deploy.sh   # Set group ownership

Master these concepts and you'll have a solid foundation in Linux security!

Shell Scripting: Automate Your Tasks

üìÖ January 28, 2026 ‚è±Ô∏è 15 min read üìä Intermediate

One of the most powerful skills in Linux is shell scripting. Scripts allow you to automate repetitive tasks, combine multiple commands, and create powerful tools.

Why Shell Scripting?

  • Automate daily tasks (backups, log cleanup)
  • Create system administration tools
  • Improve productivity exponentially
  • Create reproducible processes
  • Chain commands together intelligently

Your First Shell Script

Create a file called hello.sh:

#!/bin/bash

# This is a comment
echo "Hello, World!"
echo "Welcome to shell scripting!"

Make it executable and run it:

$ chmod +x hello.sh
$ ./hello.sh
Hello, World!
Welcome to shell scripting!

Variables

#!/bin/bash

name="Alice"
age=25
greeting="Hello, $name! You are $age years old."

echo $greeting
# Output: Hello, Alice! You are 25 years old.

Conditionals

#!/bin/bash

file="myfile.txt"

if [ -f "$file" ]; then
    echo "File exists"
else
    echo "File does not exist"
fi

Loops

For Loop

#!/bin/bash

for i in {1..5}; do
    echo "Iteration $i"
done

While Loop

#!/bin/bash

count=1
while [ $count -le 3 ]; do
    echo "Count: $count"
    count=$((count + 1))
done

Functions

#!/bin/bash

greet() {
    local name=$1
    echo "Hello, $name!"
}

greet "Alice"
greet "Bob"

Practical Example: Backup Script

#!/bin/bash

# Simple backup script
source_dir="/home/user/documents"
backup_dir="/backup/daily"
date=$(date +%Y%m%d)

echo "Starting backup..."
cp -r "$source_dir" "$backup_dir/backup_$date"
echo "Backup completed: $backup_dir/backup_$date"

Tips for Writing Better Scripts

  • Always use quotes around variables: "$var"
  • Use meaningful variable names
  • Add comments to explain your code
  • Handle errors with set -e
  • Test scripts thoroughly before automation
  • Make scripts executable with chmod +x

Shell scripting is a valuable skill that will make you significantly more productive. Start small and build up to more complex scripts!

Mastering grep Command

üìÖ December 28, 2025 ‚è±Ô∏è 9 min read üìä Beginner

The grep command is one of the most powerful tools in Linux. It allows you to search for text patterns in files and streams. Once you master grep, you'll wonder how you ever worked without it!

What is grep?

grep stands for "Global Regular Expression Print". It searches for lines matching a pattern and prints them to the output.

Basic Usage

$ grep "error" logfile.txt
$ grep "warning" /var/log/syslog

Useful Options

$ grep -i "ERROR" file.txt          # Case-insensitive
$ grep -c "error" file.txt          # Count matches
$ grep -n "error" file.txt          # Show line numbers
$ grep -v "error" file.txt          # Invert match (exclude)
$ grep -r "error" /var/log/         # Recursive search
$ grep -E "error|warning" file.txt  # Multiple patterns
$ grep "^error" file.txt            # Start of line
$ grep "error$" file.txt            # End of line

Practical Examples

Find failed logins

$ grep "Failed password" /var/log/auth.log

Count occurrences

$ grep -c "GET" access.log

Search multiple files

$ grep "TODO" *.py

Find with context

$ grep -B 2 -A 2 "error" logfile.txt   # Show 2 lines before and after

Combine with pipes

$ cat logfile.txt | grep "error" | grep "critical"
$ ps aux | grep "python"  # Find running processes

Regular Expressions with grep

$ grep "[0-9]\{3\}" file.txt              # Find 3-digit numbers
$ grep "^ERROR" file.txt                  # Line starting with ERROR
$ grep "\.log$" file.txt                  # Files ending with .log
$ grep -E "[a-z]+@[a-z]+\.[a-z]+" file   # Email addresses

Tips and Tricks

  • Use grep -i for case-insensitive searches
  • Use grep -n to quickly identify line numbers
  • Combine with other tools using pipes
  • Test patterns on small files first
  • Use grep -E for extended regex patterns

These examples barely scratch the surface. Spend time mastering grep and you'll be searching like a pro!

Disk Management in Linux

üìÖ December 10, 2025 ‚è±Ô∏è 12 min read üìä Advanced

Managing disk storage is a critical skill for Linux system administrators. Understanding partitions, filesystems, and mounting is essential for a healthy system.

Viewing Disk Information

df - Disk Space Usage

$ df -h  # Show sizes in human-readable format
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1       100G   45G   50G  47% /
/dev/sda2       200G  150G   40G  80% /home

du - Directory Size

$ du -sh /home  # Show total size of directory
45G     /home

$ du -sh /home/*  # Show size of each subdirectory
12G     /home/alice
33G     /home/bob

Partition Management

fdisk - Partition Tool

$ sudo fdisk -l  # List all partitions
$ sudo fdisk /dev/sda  # Interactive partition editor

parted - Modern Partition Tool

$ sudo parted -l  # List partitions with details
$ sudo parted /dev/sda  # Interactive mode

Filesystem Operations

Format a Partition (WARNING: Deletes Data!)

$ sudo mkfs.ext4 /dev/sda1  # Create ext4 filesystem
$ sudo mkfs.ntfs /dev/sda1  # Create NTFS filesystem

Check Filesystem

$ sudo fsck /dev/sda1  # Check for errors

Mounting and Unmounting

Mount a Partition

$ sudo mount /dev/sda1 /mnt/backup
$ mount  # View all mounted filesystems

Unmount a Partition

$ sudo umount /mnt/backup

Persistent Mounts with fstab

$ sudo nano /etc/fstab
# Add line:
/dev/sda1  /mnt/backup  ext4  defaults  0  0

Disk Usage Analysis Tools

ncdu - Interactive Disk Usage

$ ncdu /home  # Interactive directory browser

iostat - I/O Statistics

$ iostat -x 1  # Show I/O stats every 1 second

Best Practices

  • Monitor disk space regularly (aim to keep < 90% full)
  • Use separate partitions for /, /home, and /var
  • Regularly clean up old logs and temp files
  • Always backup before making partition changes
  • Use LVM for flexibility in partition management

Proper disk management ensures your system stays healthy and performs well!

User & Group Management

üìÖ November 15, 2025 ‚è±Ô∏è 11 min read üìä Intermediate

Managing users and groups is fundamental to system administration. It allows you to control access, manage permissions, and maintain a secure system.

User Management

Create a User

$ sudo useradd username
$ sudo useradd -m -s /bin/bash username  # With home dir and shell

Set User Password

$ sudo passwd username

Modify User Account

$ sudo usermod -aG sudo username  # Add to sudo group
$ sudo usermod -s /bin/zsh username  # Change shell
$ sudo usermod -d /home/newpath username  # Change home directory

Delete a User

$ sudo userdel username
$ sudo userdel -r username  # Also remove home directory

View User Information

$ id username  # Show user ID and groups
uid=1000(alice) gid=1000(alice) groups=1000(alice),4(adm),27(sudo)

$ grep username /etc/passwd  # View user entry
$ getent passwd username  # Query user database

Group Management

Create a Group

$ sudo groupadd developers

Add User to Group

$ sudo usermod -aG developers alice  # Add to secondary group
$ id alice  # Verify

View Group Information

$ getent group developers
$ cat /etc/group | grep developers

Delete a Group

$ sudo groupdel developers

Sudo Access Control

Grant Sudo Access

$ sudo usermod -aG sudo username  # Add to sudo group

Configure sudoers

$ sudo visudo  # Safely edit sudoers file

# Add line to allow specific command without password:
username ALL=(ALL) NOPASSWD: /usr/bin/systemctl

Practical Examples

Create Development Team Setup

$ sudo groupadd dev-team
$ sudo useradd -m -G dev-team alice
$ sudo useradd -m -G dev-team bob
$ sudo chown -R :dev-team /var/www/project
$ sudo chmod g+rwx /var/www/project

Restrict User Permissions

$ sudo useradd -s /usr/sbin/nologin webapp  # Doesn't allow login
$ sudo usermod -L username  # Lock user account
$ sudo usermod -U username  # Unlock user account

Best Practices

  • Create users with home directories
  • Use groups to manage permissions efficiently
  • Grant minimum necessary privileges
  • Regularly audit user and group membership
  • Disable accounts instead of deleting them when possible
  • Use strong passwords (consider ssh keys for servers)

Security Considerations

  • Limit sudo access to trusted users only
  • Use groups instead of individual permissions
  • Monitor who has access to sensitive directories
  • Regularly review and remove unnecessary accounts
  • Use password aging to enforce periodic changes

Proper user and group management is the foundation of a secure and well-organized system!
